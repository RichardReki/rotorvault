# RotorVault — Plan 3: Agent Driver (TS/viem) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or executing-plans. Steps use `- [ ]`.

**Goal:** An off-chain TypeScript agent that (1) reads the live on-chain state (RegimeGate risk-on, ApyOracle APY, vault NAV + venue positions), (2) computes a proposed FXRP venue allocation with the same `allocate()` logic as the backtest, (3) builds a `rebalance` plan and **simulates** it (dry-run default; real signing is the user's), and (4) runs the **FDC Web2Json → ApyOracle** pipeline to bring the Upshift APY on-chain.

**Architecture:** viem against Flare Coston2 (chainId 114). A read layer (public client) + a gated write layer (wallet client, only when `ENABLE_ONCHAIN_WRITE=true` and the user supplies a key). The allocation logic is a faithful TS port of `backtest/rotoredge/vault.py::allocate` (parity-tested). The FDC module builds a Web2Json attestation request for `api.upshift.finance`, polls the verifier + DA layer for the proof, and submits it to `ApyOracle.submitApy`.

**Tech Stack:** TypeScript, viem, vitest, tsx. Node 18+.

## Standing rules (carried from RangeClaw/Plan 2)
- Dry-run by default. Real `writeContract` (rebalance / submitApy) is gated behind `ENABLE_ONCHAIN_WRITE=true` **and** a user-supplied `AGENT_PRIVATE_KEY` in `.env` (gitignored). Claude never signs, never handles keys.
- All contract addresses come from `.env` (filled after Plan 2 deploy). The agent runs read/plan flows against a fork or the deployed contracts.

  id: 114,

  nativeCurrency: { name: "Coston2 Flare", symbol: "C2FLR", decimals: 18 },

### Task 2: Port `allocate()` to TS (parity test)
- [ ] Test first (`test/allocate.test.ts`) mirroring the Python tests exactly (risk-off→idle, tilt to higher APY, remainder idle, per-venue cap→idle, zero-APY→equal, negative-cap floor, clamp). Same numeric expectations as `backtest/tests/test_vault_overlay.py`.
- [ ] Implement `src/allocate.ts` as a line-for-line port of `vault.py::allocate` (same clamps, same overflow-to-idle). Output `{ fxrpExposure, venueAllocation: { firelight, upshift, idle } }`.
- [ ] `npx vitest run test/allocate.test.ts` green. Commit.

### Task 3: Signal module (read on-chain → proposed weights)
- [ ] `src/signal.ts`: `readState(client, addrs)` returns `{ riskOn, priceUsd, apyBips, nav, posFirelight, posUpshift }`; `proposeWeights(state, cfg)` = exposure (1 if riskOn else 0) × `allocate()` split by APY → `{ wFirelightBips, wUpshiftBips }`. (Firelight APY is not on-chain in v1; use ApyOracle for Upshift and a config floor for Firelight — documented LIVE-ONLY.)
- [ ] Test with a mock client returning canned reads; assert risk-off → (0,0) and risk-on tilts to the higher-APY venue. Commit.

### Task 4: Rebalance plan builder (dry-run; gated write)
- [ ] `src/rebalance.ts`: `buildPlan(client, addrs, weights)` → simulate `vault.rebalance(wFire,wUp)` via `simulateContract` (needs the agent account; use the configured account or a read-only `account` for simulation) and return the human-readable plan (targets, gas). `execute(plan)` calls `writeContract` **only if** `ENABLE_ONCHAIN_WRITE` and a wallet client exist; otherwise throws "dry-run only".
- [ ] Test that `execute` refuses without the flag. Commit. (Live simulate/execute is exercised after Plan 2 deploy — addresses in `.env`.)

### Task 5: FDC Web2Json → ApyOracle pipeline
- [ ] `src/fdc.ts`: build the Web2Json attestation request for `https://api.upshift.finance/v1/tokenized_vaults/<earnXRP>` with `postProcessJq` scaling `.reported_apy.apy` to integer bips (truncate via string-split — `floor/round` are disallowed jq builtins) and `abiSignature = {"components":[{"type":"uint256","name":"apyBips"}],...}`. POST to the verifier `…/verifier/web2/Web2Json/prepareRequest`, call `FdcHub.requestAttestation`, poll the DA layer `…/proof-by-request-round-raw`, decode to an `IWeb2Json.Proof` shape, and `ApyOracle.submitApy(proof)` (gated).
- [ ] Unit-test the request builder + jq/abi encoding (pure functions). The live round-trip needs the testnet verifier key + a deployed ApyOracle — document as a post-deploy step. Commit.

### Task 6: CLI + README
- [ ] `src/index.ts`: `tsx src/index.ts state|plan|apy`. `state` prints on-chain reads; `plan` prints the proposed rebalance (dry-run); `apy` runs the FDC pipeline (dry-run unless gated).
- [ ] `README.md`: setup, `.env`, the dry-run-vs-gated-write safety model, and that real signing/broadcast is the user's. Commit.
